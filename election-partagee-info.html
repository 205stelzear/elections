Élections Scoutes : Élections Partagées

Le but de cette fonctionnalitée

Le but des élections partagées est de permettre aux animateurs d'utiliser plusieurs appareils pour exécuter une seule élection, permettant ainsi de passer plusieurs jeunes à la fois et ainsi avoir plus de temps pour faire d'autres activités.

Ces élections sont disponible à tout ceux ayant le code pour y rentrer, peu importe où est-ce qu'il se trouve dans le monde : une élection pourrait être créée aux États-Unis et, si le code est partagé, l'élection pourrait être partagée à un appareil au Canada, si tel serait le besoin.
Ainsi, si votre groupe est à deux places en même temps, il est aussi possible grâce aux élections partagées d'avoir une élection complète.

Le serveur

Pour permettre à une élection de se faire partager, elle doit être envoyée à un serveur qui fera la gestion des requêtes entrantes et permettra de créer un code pour partager l'élection.

Ce serveur fut programmé et est maintenu par le créateur de ce site, soit Guillaume Marcoux. Pour me contacter, veuillez vous référer à mes coordonnées ci-bas.

Le serveur fut programmé pour ne pas garder les données de élections pour plus de 24 heures : pour toute opération (joindre l'élection partagée, envoyer un vote, etc), les données sur le serveur sont considérées "renouvelée" et le compteur est remis à 0. Ceci veut donc dire que dans le cas où l'élection partagée serait créée le 24 Juillet en soirée, et qu'un appareil rejoindrait l'élection avec son code le 25 Juillet en matinée et que rien ne se passe par la suite, l'élection sera supprimée du serveur le 26 Juillet en matinée.

Une option est cependant disponible à la fin de la session de vote d'une élection partagée pour supprimer les données du serveur immédiatement.

Le code de ce serveur est disponible pour ceux qui s'y connaissent et veulent savoir le fonctionnement interne du serveur : vous trouverez le code ici (GitHub).
